Breakdown

Animation

I chose Hades' audio from Walt Disney Feature Animation's Hercules because his character is extremely expressive and the way fire is used to amplify his emotions was an excellent way for me to package expressive animation and special effects into a very short segment.

Effects

I used Maya Fluids to create the fire effect. Select parts of the head geometry, such as the tips of the horns and the shoulder ridge, were duplicated to act as emitters. Their emission rates were keyframed to increase and decrease with the character's emotions. Many other attributes were altered and keyframed to make the fire more compelling, such as the buoyancy rate of the particles and their initial velocities. The input bias of the fire's incandescence attribute was also keyframed to make the fire appear blue-hot when the character is particularly angry.

Modeling

My character is loosely inspired by Lion from the game Dota 2. I used him as a reference to block out the construction of the face before adding original features such as the horns and (eventually) the flames. I created the mesh by using the create polygon tool to create the contour of the face and the major loops around the eyes, nose, and mouth. I then filled in all of the holes as effectively as possible, using a variety of tools including bridge, extrude, fill hole, interactive split tool, etc.. I tried to keep triangles to a minimum -- there were only two I couldn't eliminate: one behind the ear, and one on the crease of the nostril.

Rigging

I placed NURBS curves along all of the major curves of the face (eyebrows, eyelids, cheeks, lips, etc.) and then bound joints to all of these curves using locators. This method afforded me the ability to create many different blend shapes that would only affect a single part of the face at a time, allowing for many different dynamic combinations. Controls were then created for each blend shape using expressions

2-D Game Design

This game was built from the ground up using the C++ SDL library. It is one of the larger coding projects I have built, so adhering to strict OOP principles was a necessity. Multiple design patterns were used as well; for instance, the factory was used to instantiate the frames and artwork for the world and the characters, and the observer was used to check for collision. Game world variables, such as the locations of enemies and the screen resolution, are stored in an xml document so they can be altered without having to recompile. I am responsible for all aspects, code and artwork, of this project.

Ray Tracer

This is a fairly standard ray tracer written entirely in C. Different objects, like lights, polygons, and spheres were all stored in different files with their own header files and hit functions. I thought the standard checkboard floor layout wasn't very unique, so I tried to create a scene that looks like many different satellites rotating around a multicolored sun-like object (not that a sun would have specular reflections, but I had to show them off somehow).
